### 无隐患

**图片**：`123.jpg` (178KB)
**耗时**：1.72s

```json
{
  "name": "123.jpg",
  "analysisDetails": {
    "description": "",
    "no_split": "图片中没有安全隐患。",
    "content": [],
    "time_consuming": 1.72
  }
}
```

**判断无隐患条件**：`content` 为空数组 且/或 `no_split` 含"没有安全隐患"。

## 端点二：`/streaming/hidden_danger` — 纯文本响应

### 示例 1：隐患场景图 — 28 项（含重复）

**耗时**：10.99s

## 响应格式规律

| 字段 | 端点一 (JSON) | 端点二 (纯文本) |
|------|--------------|----------------|
| 序号 | 隐含在数组 index | `1.` `2.` ... |
| 隐患描述 | `content[].description` | `<序号>.<描述>` |
| 整改建议 | `content[].request` | `整改措施：<建议>` |
| 法规依据 | 无 | `隐患依据：<标准>`（约80%项有） |
| 无隐患标记 | `content=[]` 且 `no_split` 含"没有安全隐患" | 纯文本 `图片中没有安全隐患。` |

选择建议：需要干净结构化数据用端点一；需要法规依据条款引用用端点二。
