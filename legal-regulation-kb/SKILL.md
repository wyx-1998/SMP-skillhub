# 法规知识库 skill

## 使用场景
- 用户问：这条规定什么意思、适不适用、依据是什么
- 用户要把原始法规拆成可检索、可判定、可引用的知识单元
- 用户要把口语化问法改写成规范检索问题

## 处理原则
- 只把检索结果当证据；没有证据就说明不足以判断
- 优先保留法规原文和来源定位，不要只给结论
- 每个结论都必须能追溯到具体条款/分段/标题
- 遇到 jurisdiction、版本、生效时间缺失时先追问
- 所有检索调用都显式 UTF-8 发送，最多重试 2 次

## 标准工作流

### 1. 识别意图
- 查条款
- 判定适用
- 解释概念
- 拆解法规
- 生成问答

### 2. 规范化问题
把用户原话改写成法规检索问句，补齐：
- 主体：谁/什么对象
- 行为：做了什么/要做什么
- 条件：何时/何地/何种情形
- 例外：除外/但书/特殊情况
- 术语：同义词、上位词、行业简称

输出时同时生成：
- 规范问题
- 同义词列表
- 关键词列表（3~8 个）
- 可能的法规术语映射
- 元数据锚点（标准号、分类、文档类型、现行状态、发布日期、实施日期、发布机构、文件适用范围）

### 3. 检索知识库
必须按以下命令执行，先把规范问题写入 `query`，再发起检索。

检索时要把命中的元数据锚点一并写进 query，让检索更贴近 Dify 的元数据筛选思路：
- 标准号：优先精确匹配
- 分类：按层级或关键词扩展
- 文档类型：使用受控词表
- 现行状态：现行 / 废止 / 修订中 / 过渡期
- 发布机构：机构全称或常用简称
- 年份：按字符串前缀或完整年份匹配
- 发布日期 / 实施日期：当前字段是 string，只在用户给出精确日期或年份前缀时使用，不把它当 time 字段做 before/after 判断
- 文件适用范围：抽取对象、行业、场景关键词
- 标题：与文档标题或 `document_name` 对齐

```bash
printf '%s' '{"query":"<问题 description>"}' | \

  --data-binary @-
```

检索规则：
- 先用规范问题检索
- 如果命中弱，再用更窄的术语、同义词或条款关键词重试
- 最多重试 2 次
- 不要把原始口语直接当最终 query，必须先规范化

### 4. 把法规拆成知识单元
对每条命中结果，整理成一个独立知识单元，必须满足“单元可独立检索、可独立判断、可独立引用”。

#### 切分规则
- 以“义务 / 禁止 / 允许 / 条件 / 例外 / 但书 / 主体变化”为边界
- 一条规则里如果同时包含多个独立判断点，拆成多个单元
- 但条件与例外不能拆散到失去上下文
- 定义条款、处罚条款、程序条款分别处理，不要混成一个单元

#### 知识单元模板
- `unit_title`
- `source_title`
- `source_clause`
- `normalized_term`
- `synonyms`
- `keywords`
- `metadata_anchor`
- `matched_metadata`
- `rule_summary`
- `conditions`
- `exceptions`
- `applicability`
- `citation`
- `retrieval_score`

#### 术语标准化
- 把口语、简称、行业黑话映射为规范术语
- 保留原始表达与标准表达的对应关系
- 如果一个词有多个可能义项，列出候选，不要强行定义

#### 关键词总结
- 提炼 3~8 个高价值关键词
- 同时给出“检索关键词”和“判定关键词”
- 如果条款存在同义说法，把同义词并入关键词组

### 5. 判断适用性
把检索结果分成三类：
- **直接适用**：条款与用户场景完全吻合
- **部分相关**：能提供参考，但不能直接下结论
- **不足以判断**：缺少关键事实或检索结果不够

判断时优先检查：
- 主体是否一致
- 行为是否一致
- 条件/例外是否一致
- 适用地域、行业、版本、生效时间是否一致
- 元数据是否一致（标准号、分类、文档类型、现行状态、发布日期、实施日期、发布机构、文件适用范围）

### 6. 输出答案
答案必须按这个顺序：
1. 结论
2. 依据
3. 引用
4. 不确定性/补充条件

#### 引用要求
- 每个实质性结论都要带来源
- 引用中至少包含：文档标题、条款/段落定位、命中的原文片段
- 不要伪造条号；如果返回里没有条号，就用文档标题 + 分段信息 + 原文片段
- 如果多个来源冲突，先列冲突，再说明需要哪类信息才能判断

### 7. 需要追问时的问法
当无法判断时，优先问：
- 适用地区/法域
- 生效版本/发布日期
- 行业或场景
- 具体行为对象
- 是否要按“严格判定”还是“参考理解”处理

## 输出示例格式
```md
### 结论
...

### 依据
- ...

### 引用
- 《...》："..."
- 《...》："..."

### 说明
- 目前缺少...
```

## 质量底线
- 不凭空补条款
- 不把相似条款当作同一条
- 不把检索结果外的内容当作事实
- 不在证据不足时硬下判定
